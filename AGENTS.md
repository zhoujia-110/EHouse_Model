# E-House 几何建模工具开发目标：可视化项目配置与多剖面拼接

## 1. 项目目标

开发一个面向 E-House 钢结构 STAAD.Pro 几何建模的可视化工具。

用户不需要手写 YAML 或 JSON。用户通过图形界面完成以下工作：

1. 新建项目并输入 E-House 总尺寸；
2. 上传多个预处理后的 DXF 剖面图纸；
3. 为每张 DXF 指定所属结构平面，例如底座、顶盖、左侧墙、右侧墙、前端墙、后端墙或内部剖面；
4. 在界面中点选基准点和方向，完成 DXF 局部坐标到全局三维坐标的映射；
5. 指定构件中心面相对于外皮面的偏移；
6. 自动识别每张图纸中的构件中心线和截面标记；
7. 在单面视图中人工修正自动识别结果；
8. 将各个平面映射到统一三维坐标系；
9. 自动完成不同平面交界处的节点吸附、合并和杆件拼接；
10. 在全局视图中检查和修正三维节点与杆件；
11. 导出 nodes.csv、members.csv、warnings.csv、centerline_overlay.dxf 和 geometry.std。

## 2. 软件范围

当前阶段只处理几何建模。

包括：

* 节点；
* 杆件；
* 单面中心线识别；
* 截面标记识别；
* 多平面坐标转换；
* 多平面交界拼接；
* 人工修正；
* STAAD.Pro 几何文件导出。

不包括：

* 截面属性赋予；
* 材料；
* 支座；
* 荷载；
* 结构分析；
* 设计验算。

## 3. 用户界面要求

程序应包含以下页面：

1. 项目基本信息页；
2. 图纸上传页；
3. 剖面定位与坐标校准页；
4. 单面中心线识别页；
5. 单面人工编辑页；
6. 全局三维拼接页；
7. 全局节点/杆件修正页；
8. 导出页。

用户通过界面操作生成内部 project.json。普通用户不需要直接编辑 project.json。

## 4. 坐标体系

全局坐标体系固定为：

* X：E-House 长度方向；
* Y：E-House 高度方向；
* Z：E-House 宽度方向。

每张 DXF 图纸表示一个二维局部平面。程序需要根据用户在界面中的选择，将局部二维坐标转换为全局三维坐标。

典型映射为：

* 底座：local u → X，local v → Z，Y = 0；
* 顶盖：local u → X，local v → Z，Y = H；
* 左侧墙：local u → X，local v → Y，Z = 指定中心面；
* 右侧墙：local u → X，local v → Y，Z = 指定中心面；
* 前端墙：local u → Z，local v → Y，X = 指定中心面；
* 后端墙：local u → Z，local v → Y，X = 指定中心面；
* 内部剖面：由用户指定固定轴和局部坐标映射。

## 5. 图纸面与结构中心面

程序必须区分图纸外皮面和构件中心面。

例如，左侧墙图纸可能表示外皮面 Z=0，但实际立柱和墙梁中心线应位于 Z=50。用户应能在界面中输入“向内偏移 50 mm”，程序自动将该面上的构件坐标调整到结构中心面。

## 6. 几何识别类型

程序应支持两类构件来源：

### 6.1 轮廓构件 outline_member

适用于长条双线梁、长矩形梁、底座梁、屋面梁、墙梁和斜撑。

转换逻辑：

构件轮廓线 → 成对平行线或闭合长矩形 → 中心线 → 节点和杆件。

### 6.2 截面标记 section_marker

适用于图纸中只显示构件截面的情况，例如侧墙立柱以 SHS/RHS/H 型钢截面轮廓表示。

转换逻辑：

截面轮廓 → 形心点 → 按用户指定方向生成三维杆件 → 端点吸附到底座、顶盖或其他边界。

截面标记的轮廓边线不得直接导出为 STAAD 杆件。

## 7. 交界拼接规则

程序应支持不同结构面之间的自动拼接。

默认优先级：

1. 底座边梁中心线作为侧墙和端墙底部节点的主控对象；
2. 顶盖边梁中心线作为侧墙和端墙顶部节点的主控对象；
3. 角柱节点优先于普通墙梁节点；
4. 用户锁定节点优先级最高；
5. 自动识别的低置信度节点优先级最低。

当节点偏差小于容差时自动合并；偏差较大时标记 warning 并由用户确认。

## 8. 输出文件

程序应导出：

1. nodes.csv；
2. members.csv；
3. warnings.csv；
4. centerline_overlay.dxf；
5. global_model.json；
6. geometry.std。

geometry.std 只包含 STAAD.Pro 几何信息：

* STAAD SPACE；
* UNIT MMS KN；
* JOINT COORDINATES；
* MEMBER INCIDENCES；
* FINISH。

不输出截面、材料、支座和荷载。

## 9. 技术建议

建议使用 Python 开发。

推荐库：

* ezdxf：读取和写出 DXF；
* shapely：几何相交、投影、分割；
* numpy / scipy：几何计算和空间搜索；
* PySide6：桌面图形界面；
* pytest：单元测试。

程序架构应分层：

1. DXF 读取层；
2. 单面几何识别层；
3. 单面拓扑层；
4. 坐标映射层；
5. 全局拼接层；
6. GUI 层；
7. 导出层。

几何算法不得写死在 GUI 代码中。GUI 只能调用后端模块。
